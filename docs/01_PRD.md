# 产品需求说明 (PRD)

> AI 桌面互动伴侣 · v2.0 · 2026-03-11

---

## 1. 产品定位

**一句话描述**：常驻桌面的 AI 虚拟伴侣，能看、能听、能说、能主动互动、能帮你做事。

**目标用户**：
- 长时桌面用户（学生、程序员、创作者）
- 二次元/虚拟角色偏好用户
- 需要陪伴+轻效率助手的用户

**核心差异化**：
| 维度 | 传统 ChatBot | 本产品 |
|------|--------------|--------|
| 形态 | 聊天框 | 桌面虚拟角色 |
| 主动性 | 被动响应 | 主动触发问候/提醒 |
| 记忆 | 无/短期 | 长期记忆+关系养成 |
| 能力 | 纯对话 | 对话+截图理解+桌面操作 |

---

## 2. 功能范围

### 2.1 MVP 必做 (P0)

| 模块 | 功能 | 说明 |
|------|------|------|
| 桌面角色 | 常驻窗口 | 置顶/穿透/拖拽/多屏 |
| 对话 | 文本+语音 | 双通道输入，语音输出 |
| 主动互动 | 时间/空闲触发 | 带频控，不打扰 |
| 屏幕感知 | 截图问答 | 用户主动触发 |
| 记忆 | 短期+长期 | nanobot 复用 |
| 养成 | 亲密度/等级 | 阶段解锁 |
| 商业化 | 免费+订阅 | 权益分层 |
| 埋点 | 核心事件 | 留存/转化分析 |

### 2.2 后续版本 (P1)

- 多角色切换
- 角色商店（皮肤/语音包）
- 场景化主动策略
- 个性化推荐触发

---

## 3. 用户场景

### 场景 1：开机陪伴
```
用户开机 → 角色迎接 → "早安，今天有什么计划？"
```

### 场景 2：专注工作
```
番茄钟期间 → 角色静默监督 → 休息时提醒"该活动一下了"
```

### 场景 3：情绪陪伴
```
检测低情绪 → 主动问候 → "看起来心情不太好，想聊聊吗？"
```

### 场景 4：屏幕协助
```
用户截图 → "这是什么意思？" → AI 解释并给出建议
```

### 场景 5：桌面操作
```
用户："帮我打开 Spotify" → AI 确认后执行
```

## 4. 非功能需求

### 性能
| 指标 | 目标 |
|------|------|
| 冷启动 | < 8s |
| 常驻内存 | < 800MB |
| 文本首 token | < 1.5s |
| 语音端到端 | < 3s |

### 稳定性
- 客户端日崩溃率 < 1%
- WebSocket 自动重连
- 关键服务自动恢复

### 安全
- 本地数据默认本地存储
- 截图需显式触发
- 支持一键清空数据
- 工具执行白名单

### 兼容性
- Windows 优先首发
- 兼容主流分辨率
- 多显示器支持

## 6. 核心指标

### 北极星指标
```
NSM = DAU中有效互动用户占比 × 人均有效互动轮次
```

### 关键指标
| 指标 | MVP 目标 |
|------|----------|
| D1 留存 | ≥ 45% |
| D7 留存 | ≥ 20% |
| 14日订阅转化 | ≥ 4% |
| 主动触发接受率 | ≥ 30% |
