# AI 桌面互动伴侣 - 需求汇总

> 更新日期：2026-03-11 | 版本：v2.0

---

## 一、产品定位

### 产品形态
- **C 端桌面产品**
- **二次元虚拟人物常驻桌面（桌宠形态）**
- **非 Web 端**，原生桌面应用

### 核心体验优先级

| 优先级 | 体验 | 说明 |
|--------|------|------|
| P0 | 角色陪伴与情绪价值 | 对话、表情、动作、记忆 |
| P1 | 主动触发与习惯化 | 定时提醒、主动问候 |
| P2 | 语音/屏幕助手效率 | Do 能力（打开应用/网站/文件） |

### 目标用户
- 二次元爱好者
- 需要桌面陪伴的用户
- 希望提升桌面效率的用户

---

## 二、核心功能

### 2.1 对话能力
- 语音对话
- 文字对话
- 流式响应
- 打断恢复

### 2.2 屏幕感知
- 截图问答（用户主动触发）
- 基于截图的解释与建议

### 2.3 角色表现
- Live2D 角色渲染
- 表情与情绪联动
- 动作触发（说话、听、思考、开心等）
- 可拖拽、置顶、窗口穿透

### 2.4 记忆系统
- 短期记忆（会话内）
- 长期记忆（用户偏好、历史对话）
- 关系养成（亲密度、阶段解锁）

### 2.5 主动触发
- 定时问候（早安、晚安）
- 久坐提醒
- 特殊日期（生日、纪念日）
- 用户自定义触发

### 2.6 Do 能力（桌面工具）
| 工具 | 功能 | 权限 |
|------|------|------|
| open_app | 打开应用程序 | 需确认 |
| open_url | 打开网址 | 域名白名单 |
| open_file | 打开文件 | 需确认 |
| open_folder | 打开文件夹 | 需确认 |
| take_screenshot | 屏幕截图 | 用户触发 |
| send_notification | 发送系统通知 | 无限制 |
| get_system_info | 获取系统信息 | 只读 |

#### 2.6.1 权限与确认策略（补充）
- **执行前确认**：open_app/open_file/open_folder 需二次确认（可设置“本次会话记住”）。
- **白名单策略**：open_url 仅允许白名单域名（支持通配符），默认拒绝。
- **操作审计**：所有 Do 执行写入本地审计日志（可一键清空）。
- **最小可见性**：截图仅用户主动触发，默认不自动截图。

### 2.7 商业化
- 订阅制会员
- 功能配额（对话次数、高级模型）
- 支付集成

### 2.8 隐私与数据治理（补充）
- **数据存储位置**：明确默认数据目录与可迁移方式。
- **数据保留策略**：会话/音频/截图的默认保留周期与手动清理入口。
- **敏感信息处理**：对截图/日志中的敏感字段做脱敏或不落盘。

### 3.2 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 客户端 | Electron + TypeScript | 桌面应用框架 |
| 渲染 | Pixi.js + Live2D SDK | 角色渲染 + AudioWorklet |
| 通信 | Electron IPC | 进程间通信 |
| 后端 | Node.js (Electron Main) | Agent 核心 (TS重写) |
| S2S | Doubao Realtime API | 端到端语音 (WebSocket) |
| ASR | whisper-node | 兼容模式语音识别 |
| TTS | edge-tts | 兼容模式语音合成 |
| 打包 | electron-builder | 标准打包 |

### 3.3 模块变更说明

原 `nanobot` (Python) 方案已废弃，核心逻辑迁移至 Node.js：
- **Agent Loop**: 使用 TypeScript 重写，支持 S2S 旁路监听。
- **Memory**: 使用 SQLite/JSON 本地存储。
- **Tools**: 移植为 Node.js 实现。
